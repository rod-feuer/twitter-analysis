## AI Strategy Synthesis

The corpus is heavy on Claude Code mastery and agent engineering. Below are concrete actionable strategies extracted across the 80 AI-related bookmarks, grouped by the three angles you asked for.

### 1. Personal Workflow — adopt in your own Claude/AI usage

Most of these come from the A, D, and E categories.

1. **Build a `/documents/` folder, reference it from CLAUDE.md.** Files: `platform-docs.md`, `ICPs.md`, `styleguide.md`, `roadmap.md`, `data-reference.md`. Stop re-explaining brand/audience/standards every session. Source: [@arvidkahl](../bookmarks/2031457304328229184.md), [@BharukaShraddha](../bookmarks/2029836408232497678.md).
2. **Use plan mode + explicit "deep research" for non-trivial work.** Shift-tab → planning mode → prompt: "do deep research on best practices and known issues, using web search." Plans survive compaction better than vibe-prompted features. Source: [@arvidkahl](../bookmarks/2031457304328229184.md).
3. **Local `CLAUDE.md` near sharp edges.** Add `src/auth/CLAUDE.md`, `src/billing/CLAUDE.md`, `infra/CLAUDE.md` so Claude sees the gotchas exactly when working in those modules. Source: [@BharukaShraddha](../bookmarks/2029836408232497678.md).
4. **Enable `ENABLE_LSP_TOOL`.** Connects Claude Code to language servers — navigation in 50ms instead of 30–60s grep. Two-minute setup, 11 languages. Source: [@om_patel5](../bookmarks/2030479131222192457.md).
5. **Use Claude as a thinking partner, not a faster Google.** The YC-founder pattern: feed it raw artifacts (landing pages, transcripts, reviews), then ask:
   - "What does every successful player in this market understand that their customers never say out loud?"
   - "Show me the 3 assumptions this market is built on, and what would have to be true for each one to be wrong."
   - "Write 5 questions a world-class investor would ask to destroy this idea, then answer using only the documents."
   - "What's the strongest version of this argument and where does it still break?"
   Source: [@heyrimsha](../bookmarks/2030614728591167973.md).
6. **Karpathy's second-brain folder pattern.** Three folders: `raw/` (dump everything), `wiki/` (AI organizes), `outputs/` (AI answers questions). One CLAUDE.md schema. Every query makes the wiki better — compounds. Source: [@karpathy](../bookmarks/2039805659525644595.md), [@jumperz](../bookmarks/2039826228224430323.md), [@coreyganim](../bookmarks/2040842449384276268.md).
7. **Weekly skill audit.** Every Monday 9am, scan Linear/Notion/Slack/Gmail for repeatable processes that should become skills. Proactive AI-nativeness without making it your full-time job. Source: [@businessbarista](../bookmarks/2040173733168468177.md).
8. **Delete half your Claude setup.** Bloat hurts. Anthropic's own team explained why minimal context outperforms. Cut what's unused — long CLAUDE.mds start missing context. Source: [@itsolelehmann](../bookmarks/2036065138147471665.md), [@BharukaShraddha](../bookmarks/2029836408232497678.md).
9. **Voice + plan.md, no IDE.** Capture ideas the moment they happen as a `plan.md`. No IDE needed for many tasks. Source: [@mvanhorn](../bookmarks/2035857346602340637.md).
10. **9-step slide pipeline (Task → Context → Reference → Research → Brief → Rules → Generate → Theme → Alignment).** Stop saying "make me a deck about X." Source: [@rubenhassid](../bookmarks/2030981983992431041.md).
11. **Have Claude ask you questions before researching.** Add to prompts: "Start by using AskUserQuestion to make sure you have enough context from me before researching." Source: [@rubenhassid](../bookmarks/2030981983992431041.md), pattern echoed across [@trq212](../bookmarks/2027463795355095314.md).
12. **Free Anthropic prompting workshop.** 24 minutes, taught by Anthropic's own people. Worth more than $300 courses. Source: [@sukh_saroy](../bookmarks/2052824313997471799.md).

### 2. Build into Projects — concrete features and integrations

Most of these come from the B, C, and D categories.

1. **The Harness is the Backend.** You can outperform default Codex/Claude Code on any task by engineering the harness around it — curated prompts, tools, skills, hooks. For specialized features, build a thin task-specific harness. Source: [@mfpiccolo](../bookmarks/2049139067359568032.md), [@Vtrivedy10](../bookmarks/2052100726608781363.md), [@aparnadhinak](../bookmarks/2046980769747533830.md).
2. **Coordinator + isolated subagents (hub-and-spoke).** All communication routes through coordinator. **Subagents do NOT inherit conversation history** — every piece of information must be passed explicitly in the prompt. This is the #1 misconception. Spawn subagents in parallel via multiple Task calls in one response. Source: [@hooeem](../bookmarks/2033198345045336559.md), [@Suryanshti777](../bookmarks/2047694444787577236.md), [@trq212](../bookmarks/2027463795355095314.md).
3. **Hooks for deterministic guarantees.** Use hooks (not prompts) for anything that must happen 100% of the time:
   - PostToolUse hooks to normalize data formats (Unix→ISO timestamps, codes→strings) before the model sees them.
   - Tool-call interception hooks to block high-stakes actions (refunds > $X, compliance gates).
   Decision rule: if a single failure costs money or legal risk, use a hook. Source: [@hooeem](../bookmarks/2033198345045336559.md), [@BharukaShraddha](../bookmarks/2029836408232497678.md), [@zodchii](../bookmarks/2040000216456143002.md).
4. **2-layer event-driven agent monitoring.** Replace expensive Opus polling cron with: (a) bash pre-check that uses zero tokens when idle, (b) webhook that fires Opus only when something needs attention. ~95% token reduction. Source: [@elvissun](../bookmarks/2028671336219107687.md).
5. **Eval-driven self-improvement loop.** Define tests for your skill/agent, then run an autoresearch loop. Reported wins: Karpathy got ~11% on nanochat training-time-to-GPT-2; Hesamation took a skill from 56% → 92% in 4 rounds. Add a grader sub-agent that scores work against a rubric and feeds back to the next iteration. Source: [@karpathy](../bookmarks/2031135152349524125.md), [@Hesamation](../bookmarks/2034220679671714283.md), [@ClaudeDevs](../bookmarks/2052069324106662197.md).
6. **PRD = dataset + task function + scoring function.** Stop writing prose PRDs. The artifact a team agrees on is an eval anyone can run. Source: [@aakashgupta](../bookmarks/2039988525333176417.md).
7. **Local memory layer (BM25 + semantic).** Use QMD (Tobi Lutke's local search engine) to index your vault + Claude Code session JSONLs. Hook session export at terminal-close. Add a `/recall` skill that loads relevant context before you type. Three modes: temporal (by date), topic (BM25), graph (visualization). Source: [@ArtemXTech](../bookmarks/2028330693659332615.md).
8. **Tool-design discipline.** Tool descriptions are the primary mechanism Claude uses for selection. Scope each subagent to 4–5 tools — 18 tools degrades selection reliability. Use `tool_use` with JSON schemas (nullable for missing data, "unclear"/"other" enums) to eliminate syntax errors. Source: [@hooeem](../bookmarks/2033198345045336559.md), [@trq212](../bookmarks/2027463795355095314.md).
9. **Software factory (background coding agents).** Cloud agents that spec → PR → review → merge, invokable from Slack/phone by anyone in the org. Customer flags an issue in Slack → 15 minutes later there's a fix. Source: [@gokulr](../bookmarks/2032271386161684665.md), [@elvissun](../bookmarks/2025920521871716562.md), [@danshipper](../bookmarks/2041903948873777629.md).
10. **Three-AI code review pipeline on every PR.** Codex (edge cases / logic), Gemini Code Assist (security / scalability, free), Claude Code (validates flagged issues). Definition of done: CI passing + 3 reviews approved + screenshot if UI. Source: [@elvissun](../bookmarks/2025920521871716562.md).
11. **Progressive disclosure for context, not bigger system prompts.** Instead of dumping docs into the prompt, give Claude a link to the docs + a search tool. For specialized knowledge, a dedicated subagent (like Claude's own claude-code-guide) keeps the main agent's context clean. Source: [@trq212](../bookmarks/2027463795355095314.md).
12. **Specialized agents per role as `.md` files.** `.claude/agents/engineering.md`, `marketing.md`, `design.md`, etc. Single focused role per file. Reported 30+ working setups in the wild. Source: [@Suryanshti777](../bookmarks/2034712170647494878.md), [@gregisenberg](../bookmarks/2030680849486668229.md).
13. **Skill autoresearch.** Apply Karpathy's autoresearch loop to skills themselves — define tests for a skill, let an agent iterate on the skill file across rounds. Source: [@Hesamation](../bookmarks/2034220679671714283.md).

If you hire or interview, expect this shift; if you're building dev tools, this is a market signal. Source: [@btaylor](../bookmarks/2046996207898366248.md).

One-shot analysis: I read every bookmark markdown (and full JSON `plain_text` for AI-categorized articles), derived the taxonomy from the corpus, and synthesized strategies across them. No code was added to the project. If the corpus grows substantially or you want this re-run on every sync, the next step would be a `categorize.py` script that calls the Claude API — but with 99 bookmarks one-shot is the right tool.
